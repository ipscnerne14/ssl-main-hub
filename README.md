# ssl-main-hub

## 项目简介

`ssl-main-hub` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何单个域名或具体网站，而是作为一个统一的索引与存放中心，将不同来源的静态页面进行集中管理，便于后续浏览、分享与维护。

## 目录结构

```
ssl-main-hub/
├── index.html          # 仓库主入口页面，列出所有归档页面链接
├── pages/              # 存放各独立 HTML 页面的文件夹
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源文件夹（CSS、JS、图片等）
│   ├── style.css
│   └── script.js
└── README.md           # 本文件
```

- `index.html`：仓库的导航页面，建议通过该页面访问所有归档内容。
- `pages/`：所有独立 HTML 页面均放置在此目录下，每个文件代表一个独立的归档页面。
- `assets/`：存放各页面可能共用的样式、脚本及静态资源，减少重复。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，彼此之间无直接关联。每个页面可以包含自身所需的全部样式与脚本，亦可引用 `assets/` 中的公共资源。

- 页面命名建议简洁、有意义，避免特殊字符。
- 每个页面应尽量保持自包含，便于单独部署或分享。
- 如需更新或替换某个页面，请直接修改对应文件，并确保 `index.html` 中的链接同步更新。

## 维护说明

- **添加新页面**：在 `pages/` 下新建 HTML 文件，并在 `index.html` 的列表中添加对应链接。
- **更新已有页面**：直接编辑 `pages/` 下的对应文件。
- **删除页面**：移除 `pages/` 中的文件，并同步更新 `index.html` 中的链接。
- **公共资源**：如需修改全局样式或脚本，请编辑 `assets/` 下的文件，并确保所有引用该资源的页面兼容。

## 贡献与使用

本仓库主要用于个人或团队的静态页面归档与发布。欢迎通过 Pull Request 提交改进或建议，但请注意保持仓库的通用性与简洁性。

本仓库内容按需使用，不附带任何特定许可声明。如引用第三方资源，请遵守其原始许可要求。
